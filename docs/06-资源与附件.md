# 06 — 资源与附件

## 资源目录

将图片、音频等静态文件放在插件根目录的 `assets/` 下：

```
my-plugin/
├── runtime/
│   └── bootstrap.js
└── assets/
    ├── welcome.png
    └── sounds/
        └── ding.mp3
```

## 路径解析规则

宿主在处理插件提供的资源路径时，按以下规则解析：

| 路径格式 | 行为 | 示例 |
|---------|------|------|
| 相对路径 | 自动拼接插件根目录 | `"assets/welcome.png"` → `/data/.../my-plugin/assets/welcome.png` |
| 绝对路径 | 原样使用 | `"/sdcard/Pictures/photo.jpg"` |
| `http://` / `https://` | 网络资源，原样使用 | `"https://example.com/img.png"` |
| `file://` | 原样使用 | `"file:///sdcard/photo.jpg"` |
| `content://` | Android content URI，原样使用 | `"content://media/..."` |
| `base64:` | Base64 数据，原样使用 | `"base64:iVBOR..."` |
| `plugin://` | 插件协议 URI，原样使用 | `"plugin://..."` |

> **推荐使用相对路径**。宿主会自动解析到正确的绝对位置，插件无需关心实际安装目录。

## 在命令回复中使用附件

```javascript
handler(event) {
  event.replyResult({
    text: "这是一张图片：",
    attachments: [
      {
        source: "assets/welcome.png",   // 相对路径，自动解析
        mimeType: "image/png",          // 可选，帮助宿主判断类型
        label: "欢迎图",                 // 可选，等价于 altText / fileName
      },
    ],
  });
}
```

### attachment 对象字段

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `source` | string | ✅ | 资源路径（优先级最高） |
| `uri` | string | ❌ | `source` 的别名 |
| `assetPath` | string | ❌ | `source` 的别名 |
| `path` | string | ❌ | `source` 的别名 |
| `mimeType` | string | ❌ | MIME 类型，如 `"image/jpeg"` |
| `label` | string | ❌ | 显示标签 |
| `altText` | string | ❌ | `label` 的别名 |
| `fileName` | string | ❌ | `label` 的别名 |

> `source` / `uri` / `assetPath` / `path` 按优先级取第一个非空值。
> `label` / `altText` / `fileName` 按优先级取第一个非空值。

## 在 LLM 钩子中使用附件

`on_decorating_result` 阶段的 `result` 对象支持附件操作：

```javascript
hostApi.registerLlmHook({
  hook: "on_decorating_result",
  key: "my-plugin.add-image",
  priority: 100,
  metadata: {},
  handler(payload) {
    const result = payload && payload.result;
    if (!result) return;

    // 追加一个附件
    result.appendAttachment({
      uri: "assets/decoration.png",
      mimeType: "image/png",
    });

    // 或者替换所有附件
    // result.replaceAttachments([
    //   { uri: "assets/new.jpg", mimeType: "image/jpeg" },
    // ]);

    // 或者清除所有附件
    // result.clearAttachments();
  },
});
```

### result 附件方法

| 方法 | 说明 |
|------|------|
| `result.appendAttachment({ uri, mimeType? })` | 追加一个附件 |
| `result.replaceAttachments([ { uri, mimeType? } ])` | 替换为新的附件列表 |
| `result.clearAttachments()` | 清除所有附件 |

> **注意**：LLM 钩子中附件的字段名是 `uri`（不是 `source`），与命令回复略有不同。

## 常见 MIME 类型

| 类型 | mimeType |
|------|----------|
| JPEG 图片 | `image/jpeg` |
| PNG 图片 | `image/png` |
| GIF 动图 | `image/gif` |
| WebP 图片 | `image/webp` |
| MP3 音频 | `audio/mpeg` |
| MP4 视频 | `video/mp4` |
