# 05 — 配置与设置

插件有两套配置机制，服务于不同场景：

| 文件 | 用途 | 读取方式 |
|------|------|----------|
| `_conf_schema.json` | 定义插件的可配置项及默认值 | `hostApi.getSettings()` |
| `schemas/settings-schema.json` | 定义宿主设置 UI 的渲染结构 | 宿主自动读取并渲染 |

---

## _conf_schema.json — 静态配置定义

每个键就是一个配置项，值是描述对象：

```json
{
  "greeting_prefix": {
    "type": "string",
    "description": "问候语前缀",
    "hint": "命令回复时显示在名字前面的文本",
    "default": "你好",
    "section": "general"
  },
  "enable_echo": {
    "type": "bool",
    "description": "启用复读功能",
    "hint": "开启后正则匹配到的消息会被原样复读",
    "default": true,
    "section": "general"
  },
  "max_retries": {
    "type": "number",
    "description": "最大重试次数",
    "hint": "失败后自动重试的最大次数",
    "default": 3,
    "section": "advanced"
  },
  "llm_footer": {
    "type": "text",
    "description": "LLM 回复后缀",
    "hint": "追加到每条 LLM 回复末尾的文本，留空则不追加",
    "default": "",
    "section": "advanced"
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | string | 见下方类型表 |
| `description` | string | 配置项简述 |
| `hint` | string | 更详细的提示文本 |
| `default` | any | 默认值 |
| `section` | string | 分组标识，用于 UI 归类 |

### 支持的 type 值

| type | 说明 | 默认值类型 | 备注 |
|------|------|-----------|------|
| `"string"` | 单行文本 | string | |
| `"text"` | 多行文本 | string | |
| `"bool"` | 布尔开关 | boolean | |
| `"int"` | 整数 | number | |
| `"float"` | 浮点数 | number | |
| `"number"` | 浮点数（`"float"` 的别名） | number | 推荐使用 |

> `"number"` 和 `"float"` 完全等价，`"number"` 更直观，推荐优先使用。

### 在代码中读取

```javascript
const settings = hostApi.getSettings() || {};
const prefix = settings.greeting_prefix ?? "你好";
const enabled = settings.enable_echo ?? true;
const retries = settings.max_retries ?? 3;
```

> `getSettings()` / `readSettings()` / `getPluginSettings()` / `getConfig()` 四个方法等价。

> 首次运行时用户可能尚未保存配置，返回值为 `{}`。**务必用 `??` 或 `||` 提供默认值。**

---

## schemas/settings-schema.json — 设置界面 UI

控制宿主 App 中"插件设置"页面的 UI 渲染。不影响配置读取逻辑。

```json
{
  "title": "Hello World 设置",
  "sections": [
    {
      "sectionId": "general",
      "title": "基础设置",
      "fields": [
        {
          "fieldType": "text_input",
          "fieldId": "greeting_prefix",
          "label": "问候语前缀",
          "placeholder": "例如：你好",
          "defaultValue": "你好"
        },
        {
          "fieldType": "toggle",
          "fieldId": "enable_echo",
          "label": "启用复读功能",
          "defaultValue": true
        },
        {
          "fieldType": "select",
          "fieldId": "mode",
          "label": "工作模式",
          "defaultValue": "auto",
          "options": [
            { "value": "auto", "label": "自动" },
            { "value": "manual", "label": "手动" }
          ]
        }
      ]
    },
    {
      "sectionId": "advanced",
      "title": "高级设置",
      "fields": [
        {
          "fieldType": "text_input",
          "fieldId": "llm_footer",
          "label": "LLM 回复后缀",
          "placeholder": "留空则不追加",
          "defaultValue": ""
        }
      ]
    }
  ]
}
```

### 顶层结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 设置页面标题 |
| `sections` | array | 分组列表 |

### section 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `sectionId` | string | 分组唯一标识 |
| `title` | string | 分组标题 |
| `fields` | array | 字段列表 |

### field 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `fieldType` | string | 控件类型 |
| `fieldId` | string | 字段 ID（应与 `_conf_schema.json` 中的 key 对应） |
| `label` | string | 显示标签 |
| `placeholder` | string | 占位文本（仅 `text_input`） |
| `defaultValue` | any | 默认值 |
| `options` | array | 选项列表（仅 `select`） |

### 支持的 fieldType

| fieldType | 渲染为 | defaultValue 类型 |
|-----------|--------|------------------|
| `toggle` | 开关 | boolean |
| `text_input` | 文本输入框 | string |
| `select` | 下拉选择 | string（对应 option.value） |

### 两套配置文件的关系

- `_conf_schema.json` 定义**数据**：有哪些配置项、类型、默认值
- `settings-schema.json` 定义**UI**：宿主如何渲染设置界面
- `fieldId` 应与 `_conf_schema.json` 中的 key 一一对应
- 两者可以独立存在：只有 `_conf_schema.json` 时插件仍可读取配置（但无 UI）；只有 `settings-schema.json` 时宿主会渲染 UI
- 推荐两者都提供，保持 `fieldId` 和配置 key 一致

### 重要：当 fieldId 与宿主核心设置键重叠时

如果插件的 `_conf_schema.json` 中的某个 key 与宿主核心设置（`settings-schema.json`）中的 `fieldId` 重叠，宿主会自动处理：
- 在"运行时设置"界面中显示该字段（来自核心设置 UI）
- 在"插件配置"界面中隐藏该字段（避免重复）
- 保存时写入核心设置存储
- 读取时从核心设置中获取值

> 一般插件开发不需要关心这个机制，除非你有意复用宿主已有的设置键。
