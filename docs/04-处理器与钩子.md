# 04 — 处理器与钩子

所有处理器在 `bootstrap(hostApi)` 中通过 `hostApi.registerXxxHandler()` 注册。

---

## 命令处理器

```javascript
hostApi.registerCommandHandler({
  key: "my-plugin.greet",          // 全局唯一 key
  command: "hello",                // 命令名（用户输入 /hello）
  aliases: ["hi", "你好"],         // 别名列表
  groupPath: [],                   // 命令分组路径，留空即顶级命令
  priority: 0,                     // 数字越小越先执行
  filters: [],                     // 过滤器（高级），通常留空
  metadata: {                      // 自定义元数据，不影响逻辑
    description: "打招呼命令",
  },
  handler(event) {
    // 处理逻辑
  },
});
```

### 命令事件对象 `event`

| 属性 | 类型 | 说明 |
|------|------|------|
| `eventId` | string | 事件唯一 ID |
| `platformAdapterType` | string | 平台类型（如 `"qq"`） |
| `messageType` | string | 消息类型 |
| `conversationId` | string | 会话 ID |
| `senderId` | string | 发送者 ID |
| `timestampEpochMillis` | number | 时间戳（毫秒） |
| `rawText` | string | 原始消息文本 |
| `workingText` | string | 经前置处理后的文本 |
| `normalizedMentions` | string[] | @提及列表 |
| `extras` | object | 附加信息 |
| `stage` | string | 当前阶段，固定 `"Command"` |
| `commandPath` | string[] | 匹配到的命令路径 |
| `args` | string[] | 命令参数数组，如 `/hello 世界 你好` → `["世界", "你好"]` |
| `matchedAlias` | string | 实际匹配到的命令名或别名 |
| `remainingText` | string | 命令名之后的完整原始文本 |
| `invocationText` | string | 完整触发文本 |

### 命令事件方法

| 方法 | 说明 |
|------|------|
| `event.replyResult(payload)` | 结构化回复，payload 为 string 或 `{ text, attachments? }` |
| `event.replyText(text)` | 纯文本回复 |
| `event.stopPropagation()` | 阻止后续同命令的其它处理器 |

> `sendResult` / `reply` / `respond` 等价于 `replyResult`；`sendText` / `respondText` 等价于 `replyText`

### replyResult 的 payload 格式

```javascript
// 方式 1：纯字符串
event.replyResult("你好！");

// 方式 2：带附件的对象
event.replyResult({
  text: "这是回复文本",
  attachments: [
    {
      source: "assets/photo.jpg",       // 相对路径（自动解析到插件目录）
      mimeType: "image/jpeg",           // 可选
      label: "照片",                     // 可选，等价于 altText / fileName
    },
  ],
});
```

附件的 `source` 字段支持以下格式（优先级：`source` > `uri` > `assetPath` > `path`）：
- 相对路径 → 自动解析为 `插件根目录/相对路径`
- 绝对路径 → 原样使用
- `http://` / `https://` → 网络资源
- `file://` / `content://` / `base64:` / `plugin://` → 原样使用

## 消息处理器

在命令匹配**之前**触发，可用于日志记录、消息预处理等。

```javascript
hostApi.registerMessageHandler({
  key: "my-plugin.msg-logger",
  priority: 0,
  filters: [],
  metadata: {},
  handler(event) {
    // event 与命令事件类似，但无 args / commandPath 等命令专有字段
    hostApi.log("INFO", `收到消息: ${event.rawText}`);
  },
});
```

### 消息事件对象 `event`

| 属性 | 类型 | 说明 |
|------|------|------|
| `eventId` | string | 事件唯一 ID |
| `platformAdapterType` | string | 平台类型 |
| `messageType` | string | 消息类型 |
| `conversationId` | string | 会话 ID |
| `senderId` | string | 发送者 ID |
| `timestampEpochMillis` | number | 时间戳 |
| `rawText` | string | 原始消息 |
| `workingText` | string | 处理后文本 |
| `normalizedMentions` | string[] | @列表 |
| `extras` | object | 附加信息 |
| `stage` | string | `"AdapterMessage"` |

方法：`event.stopPropagation()` — 阻止后续消息处理器和命令匹配。

## 正则处理器

匹配符合正则表达式的消息。

```javascript
hostApi.registerRegexHandler({
  key: "my-plugin.echo",
  pattern: "^echo[：:]\\s*(.+)",   // 正则字符串（非 RegExp 对象）
  flags: "i",                       // 正则标志
  priority: 0,
  filters: [],
  metadata: {},
  handler(event) {
    const captured = event.groups[0]; // 第一个捕获组
    hostApi.log("INFO", `匹配到: ${captured}`);
    event.stopPropagation();
  },
});
```

### 正则事件对象 `event`

除了消息事件的公共字段外，还有：

| 属性 | 类型 | 说明 |
|------|------|------|
| `patternKey` | string | 匹配到的 key |
| `matchedText` | string | 完整匹配文本 |
| `groups` | string[] | 捕获组数组（`groups[0]` = 第一个 `()`） |
| `namedGroups` | object | 命名捕获组 `(?<name>...)` |

方法：`event.stopPropagation()`

## LLM 钩子

拦截 AI 对话流水线的各个阶段。

```javascript
hostApi.registerLlmHook({
  hook: "on_decorating_result",     // 钩子阶段
  key: "my-plugin.llm-decorate",
  priority: 100,                    // 数字越小越先执行
  metadata: {},
  handler(payload) {
    // payload 为单个对象，结构因 hook 阶段而异
  },
});
```

### 五个钩子阶段

| hook | 触发时机 | payload 结构 | 描述 |
|------|----------|-------------|------|
| `on_waiting_llm_request` | LLM 请求排队时 | 扁平对象（含 event 字段） | 观察即将发起的请求 |
| `on_llm_request` | LLM 请求发出前 | `{ event, request }` | **可修改** request |
| `on_llm_response` | LLM 返回后 | `{ event, response }` | 观察原始 LLM 回复 |
| `on_decorating_result` | 回复发送前 | `{ event, result }` | **可修改** result 文本和附件 |
| `after_message_sent` | 消息已发送 | `{ event, view }` | 可补发消息 |

### on_llm_request — 修改请求

```javascript
handler(payload) {
  const { event, request } = payload;
  // request 可读属性:
  //   requestId, conversationId, messageIds, llmInputSnapshot,
  //   availableProviderIds, availableModelIdsByProvider,
  //   selectedProviderId, selectedModelId,
  //   systemPrompt, messages, temperature, topP, maxTokens,
  //   streamingEnabled, metadata

  // 修改方法:
  request.appendSystemPrompt("额外指令...");
  request.replaceSystemPrompt("完全替换系统提示");
}
```

### on_llm_response — 观察回复

```javascript
handler(payload) {
  const { event, response } = payload;
  // response 可读属性:
  //   requestId, providerId, modelId, text, markdown,
  //   finishReason, metadata,
  //   usage: { promptTokens, completionTokens, totalTokens,
  //            inputCostMicros, outputCostMicros, currencyCode },
  //   toolCalls: [{ toolName, arguments, metadata }]
}
```

### on_decorating_result — 修改最终回复

这是最常用的钩子，在 AI 回复发送给用户之前做最后修改。

```javascript
handler(payload) {
  const { event, result } = payload;
  // result 可读属性:
  //   requestId, conversationId, text, markdown,
  //   attachments: [{ uri, mimeType }],
  //   attachmentMutationIntent, shouldSend, isStopped

  // 修改方法:
  result.appendText("\n— 由插件追加");
  result.replaceText("完全替换文本");
  result.clearText();

  result.appendAttachment({ uri: "assets/img.png", mimeType: "image/png" });
  result.replaceAttachments([{ uri: "assets/a.jpg", mimeType: "image/jpeg" }]);
  result.clearAttachments();

  result.setShouldSend(false);    // 阻止发送
  result.stop();                  // 阻止后续钩子
}
```

### after_message_sent — 发送后回调 + 补发

```javascript
handler(payload) {
  const { event, view } = payload;
  // view 只读属性:
  //   requestId, conversationId, sendAttemptId,
  //   platformAdapterType, platformInstanceKey,
  //   sentAtEpochMs, deliveryStatus, receiptIds,
  //   deliveredEntries: [{ entryId, entryType, textPreview, attachmentCount }],
  //   deliveredEntryCount,
  //   usage: { promptTokens, completionTokens, ... }

  // view 操作属性:
  //   canSendFollowup  — boolean，是否支持补发
  //   sendFollowup(text, attachments) — 补发一条消息

  // 补发示例:
  if (view.canSendFollowup && typeof view.sendFollowup === "function") {
    const result = view.sendFollowup("补发文本", [
      { uri: "assets/extra.png", mimeType: "image/png" },
    ]);
    // result: { success: boolean, receiptIds: string[], errorSummary: string }
  }
}
```

> **注意**：`canSendFollowup` 在 QQ 等外部平台下为 `true`，在应用内聊天下为 `false`。

## 生命周期钩子

```javascript
// 快捷写法
hostApi.onPluginLoaded({
  key: "my-plugin.loaded",
  handler() {
    hostApi.log("INFO", "插件已加载");
  },
});

hostApi.onPluginUnloaded({
  key: "my-plugin.unloaded",
  handler() {
    hostApi.log("INFO", "插件已卸载");
  },
});

hostApi.onPluginError({
  key: "my-plugin.error",
  handler(payload) {
    hostApi.log("ERROR", "插件出错", payload || {});
  },
});

// 等价的完整写法
hostApi.registerLifecycleHandler({
  hook: "on_plugin_loaded",
  key: "my-plugin.loaded",
  priority: 100,
  metadata: {},
  handler() { /* ... */ },
});
```

## priority 说明

- 所有处理器和钩子都有 `priority` 字段
- **数字越小越先执行**
- 推荐范围：`0`（高优先）~ `200`（低优先）
- 同优先级的执行顺序不保证
